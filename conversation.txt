     PokemonSimulator 本轮改动计划

     改动一：动画回退修复 — "先对后演"

     问题

     10秒轮询期间，新数据到达时 _tickUpdate 正在逐行 splice 做滑动动画。_tick++
     中断旧动画后，filtered 留在中间态（部分行已移动），新动画从这个错误位置开始
     ，产生"滑上去又滑回来"的回退效果。

     方案：即时正确 + 异步重演

     新数据到 → 立刻 filtered = target（保证数据始终正确）
              → 从旧顺序生成重演队列 → 逐帧滑到新顺序（纯视觉）
              → 重演期间 filtered 不变，不响应新数据
              → 重演结束 → 等待下一轮数据

     核心改动在 DataTable.vue 的 _tickUpdate 函数：

     async function _tickUpdate(target) {
       const tick = ++_tick
       const idKey = props.idKey || 'id'
       const old = [...filtered.value]  // 快照旧顺序

       // 1. 立即设置正确数据（不等动画）
       filtered.value = target
       applySort()

       // 2. 异步重演：先赋值新值，再逐行滑到新位置
       const moved = computeMoves(old, filtered.value, idKey)  // 对比新旧位置
       if (moved.length === 0) return

       const arr = [...filtered.value]
       _display.value = arr  // 用 _display 做视觉缓冲
       for (const { id, to } of moved) {
         if (tick !== _tick) { _display.value = null; return }
         const cur = arr.findIndex(r => r?.[idKey] === id)
         if (cur === -1 || cur === to) continue
         arr.splice(to, 0, arr.splice(cur, 1)[0])
         _display.value = [...arr]
         await new Promise(r => setTimeout(r, 1000))
       }
       if (tick === _tick) _display.value = null  // 重演结束
     }

     关键区别：

     ┌────────────┬────────────────────┬─────────────────────────────┐
     │            │        当前        │           改造后            │
     ├────────────┼────────────────────┼─────────────────────────────┤
     │ 数据准确性 │ 动画中途被中断会乱 │ 始终正确（先设值）          │
     ├────────────┼────────────────────┼─────────────────────────────┤
     │ 中断处理   │ 残留中间态         │ 无影响（filtered 不受干扰） │
     ├────────────┼────────────────────┼─────────────────────────────┤
     │ 视觉效果   │ 同时做动画和排序   │ 异步重演，不阻塞数据        │
     └────────────┴────────────────────┴─────────────────────────────┘

     附加优化：最大重演时间

     如果挪动的行太多（比如 20 行），重演需要 20s，但轮询只有
     10s。加一个截断：超过 maxReplaySec 秒后直接 _display = null 跳到最终态。
